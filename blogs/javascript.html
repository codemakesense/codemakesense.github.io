# apply 和 call

一、什么是 call、apply 方法 每个函数都包含两个非继承而来的方法：apply()和 call()、这两个方法的用途都是在特定的作用域中调用函数，实际上等于设置函数体内的 this 对象的值。

二、aplly() apply 方法接收两个参数：一个是其中运行函数的作用域，另一个是参数数组。其中，第二个参数可以是 Array 的实例，也可以是 arguments 对象。

function sum(num1, num2) {
  return num1 + num2
}

function callSum1(sum1, num2) {
  return sum.apply(this, arguments)
}

function callSum2(num1, num2) {
  return sum.apply(this, [num1, num2])
}

alert(callSum1(10,10))    //20
alert(callSum2(10,10))    //20

# 闭包的理解

JS 中的闭包，按自己消化的理解就是：由于存在作用域链，使得父作用域无法访问子作用域的内容，而子作用域可以沿着作用域链访问父作用域乃至全局的内容；

而 JS 的作用域是以函数为分界的，因此闭包也可以理解为可以访问另一个函数作用域的变量的函数。

那么闭包的作用是什么呢：

可以访问函数内部的变量 就像下面的例子一样：

function Person() {
  var name = 'Jack'
  function getName() {
    console.log(name)
  }
  return getName
}

let persons = new Person()
persons()   // Jack

原本由于作用域链，在Person函数外部是无法访问name属性的，通过闭包可以在函数内部新构造一个函数获得name属性，并将这个函数return出去。

可以使变量的值长期保存在内存中，生命周期比较长（在页面关闭前） 因此不能滥用闭包，否则会造成内存泄漏导致网页性能问题。应当在闭包使用完成之后立即释放资源，将引用变量指向null进行销毁。可以像下面的例子这么做：

function assignHandler() {
  var element = document.getElementById('someId')
  var id = element.id
  element.onclick = function() {
    console.log(id)
  }
  element = null
}

当以原型模式构造一个函数，如：

function Person() {

}

prototype（原型）属性，即调用构造函数而创建的原型对象，可以让所有对象实例共享它所包含的属性和方法。免去了在构造函数中定义实例对象的信息，而是直接将信息添加到原型对象中。 同时，对对象实例的属性进行赋值，只能覆盖原型对象的属性，不能影响它的属性

var person1 = new Person()
var person2 = new Person()

alert(person1.name)    // 提示"Jack"

person1.name = "Tom"

alert(person1.name)    // 提示"Tom"
alert(person2.name)    // 提示"Jack"

# 原型链

在使用原型链过程中，如果同一个原型对象有不同的实例对象，那么无法在不影响其他实例对象的情况下，给这个原型对象的构造函数传递参数，情况如下：

function Case() {
  this.color = ["red","green"]
}

function OtherCase() {}

OtherCase.prototype = new Case()

var instance1 = new OtherCase()
instance1.color.push("blue")
var instance2 = new OtherCase()

console.log(instance2.color)    //"red,green,blue"
console.log(instance1.color)    //"red,green,blue"

就如上面所示，可能目的只是往 instance1 的原型链上的 color 属性插入一个"blue"字符串，但是同时也影响了共用同一个原型对象的 instance2。

解决办法就是使用 call()或者 apply()方法在函数内部调用原型对象的构造函数。

function Case() {
  this.color = ["red","green"]
}

function OtherCase() {
  Case.apply(this)    //继承了Case
}

var instance1 = new OtherCase()
instance1.color.push("blue")
var instance2 = new OtherCase()

console.log(instance2.color)    //“red,green”
console.log(instance1.color)    //"red,green,blue"

# 组合构造函数

虽然原型模式构造函数中的所有属性是被很多实例共享的，但是在使用不同实例的时候，我们还是希望不同实例的某些属性具备独特性，这个问题下面例子可以体现：

function School() {}

School.prototype = {
  name : "qinghua",
  position : "beijing",
  otherschool : ["beida", "renda"],
  sayName : function() {
    alert(this.name)
  }
}

var school1 = new School()
var school2 = new School()

alert(school1.otherschool) //"beida,renda"
alert(school2.otherschool) //"beida,renda"

school1.otherschool.push("beishida")

alert(school1.otherschool) //"beida,renda,beishida"
alert(school2.otherschool) //"beida,renda,beishida"
alert(school1.otherschool == school2.otherschool) //true

当同时创建了 School.prototype 的两个实例：school1 和 school2，他们共享了原型的所有属性，这是正常的。但是当往 school1.otherschool 属性更新数组内容时，可能本意只是想在 school1 上更新；但是由于 otherschool 并非存在于 school1 中，而是 School.prototype，因此修改数组字符串时会导致 school2 的 otherschool 属性也同时被修改。

想避免这种情况，可以构造函数模式和原型模式组合使用，即让通用的属性由原型模式存放，构造函数模式用于定义实例属性。还是上面的例子：

function School(name, position, otherschool)  {
  this.name = name,
  this.position = position,
  this.otherschool = ["beida", "renda"],
}

School.prototype = {
  sayName : function() {
  alert(this.name)
}
}

var school1 = new School("qinghua", "beijing")
var school2 = new School("wuda", "wuhan")

school1.otherschool.push("beishida")

alert(school1.otherschool)    //"beida,renda,beishida"
alert(school2.otherschool)    //"beida,renda"

alert(school1.otherschool == school2.otherschool)   //false
alert(school1.sayName == school2.sayName)   //true
